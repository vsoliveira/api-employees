# Testes de Estresse com k6

## Visão Geral

O cenário de k6 estressa a API de funcionários sobre a base de dados e exporta métricas para o Prometheus, que depois são visualizadas no Grafana.

## Cenário Principal

O script principal é `tests/k6/employees-api-stress.js`.

Ele mistura duas cargas:

- tráfego de leitura paginada em `GET /api/v1/employees`
- escrita que cria e remove registros em sequência

## Wrapper Recomendado

```bash
./scripts/run-k6-prometheus.sh employees-dashboard-smoke
```

O primeiro argumento vira a tag `testid`. Se ele não for informado, o wrapper gera um valor com base no timestamp atual.

## Stack Dev Necessária

Antes de rodar o k6, inicie o ambiente de desenvolvimento:

```bash
cd docker/dev
docker compose up --build -d
```

O script espera:

- URL base da API: `http://localhost:8080/api`
- API key padrão: `dev-api-key`
- Endpoint de remote-wwrite do Prometheus: `http://localhost:9090/api/v1/write`

## Flags de Exportação para o Prometheus

| Configuração | Finalidade |
| --- | --- |
| `K6_PROMETHEUS_RW_SERVER_URL` | Endpoint de remote write do Prometheus |
| `K6_PROMETHEUS_RW_TREND_STATS` | Agregações de tendência publicadas como séries |
| `K6_PROMETHEUS_RW_STALE_MARKERS=true` | Marca as séries como stale ao final do teste |
| `-o experimental-prometheus-rw` | Habilita a saída de remote write do Prometheus |
| `--tag testid=<run-id>` | Isola uma execução específica no Grafana |

## Execução Manual em Contêiner Dockerizado

```bash
docker run --rm \
  --network host \
  -v "$PWD:/work" \
  -w /work \
  -e BASE_URL=http://localhost:8080/api \
  -e API_KEY=dev-api-key \
  -e K6_PROMETHEUS_RW_SERVER_URL=http://localhost:9090/api/v1/write \
  -e K6_PROMETHEUS_RW_TREND_STATS=p(95),p(99),avg,max \
  -e K6_PROMETHEUS_RW_STALE_MARKERS=true \
  grafana/k6:latest run -o experimental-prometheus-rw --tag testid=<unique-run-id> tests/k6/employees-api-stress.js
```

## Overrides Úteis

- `BROWSE_RATE`: taxa estável de leitura paginada por segundo
- `WRITE_VUS`: quantidade de usuários virtuais escritores
- `PAGE_SIZE`: tamanho de página das listagens
- `BROWSE_STAGE_ONE_DURATION`, `BROWSE_STAGE_TWO_DURATION`, `BROWSE_STAGE_THREE_DURATION`: ramp-up, steady-state e ramp-down
- `WRITE_DURATION`: duração total do cenário de escrita

## Fluxo de Dashboard

1. Rode o wrapper ou o comando manual do k6.
2. Abra o Grafana em `http://localhost:3000`.
3. Abra o dashboard `k6 Stress Overview`.
4. Filtre o dashboard pelo `testid` da execução.
