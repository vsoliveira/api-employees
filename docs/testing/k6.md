# k6 Stress Testing

## Overview

The k6 scenario in this repository exercises the employee API against the seeded dataset and exports metrics to Prometheus for visualization in Grafana.

## Main Scenario

The primary script is `tests/k6/employees-api-stress.js`.

It mixes two workloads:

- read-heavy paginated `GET /api/v1/employees` traffic
- write churn that creates and then deletes employee records

## Recommended Wrapper

```bash
./scripts/run-k6-prometheus.sh employees-dashboard-smoke
```

The first argument becomes the `testid` tag. If omitted, the wrapper generates one from the current timestamp.

## Required Dev Stack

Before running k6, start the development environment:

```bash
cd docker/dev
docker compose up --build -d
```

The script expects:

- API base URL: `http://localhost:8080/api`
- default API key: `dev-api-key`
- Prometheus remote-write endpoint: `http://localhost:9090/api/v1/write`

## Prometheus Export Flags

| Setting | Purpose |
| --- | --- |
| `K6_PROMETHEUS_RW_SERVER_URL` | Prometheus remote-write endpoint |
| `K6_PROMETHEUS_RW_TREND_STATS` | Trend aggregations published as series |
| `K6_PROMETHEUS_RW_STALE_MARKERS=true` | Marks exported series stale after the run |
| `-o experimental-prometheus-rw` | Enables Prometheus remote write output |
| `--tag testid=<run-id>` | Isolates one run inside Grafana dashboards |

## Manual Dockerized Run

```bash
docker run --rm \
  --network host \
  -v "$PWD:/work" \
  -w /work \
  -e BASE_URL=http://localhost:8080/api \
  -e API_KEY=dev-api-key \
  -e K6_PROMETHEUS_RW_SERVER_URL=http://localhost:9090/api/v1/write \
  -e K6_PROMETHEUS_RW_TREND_STATS=p(95),p(99),avg,max \
  -e K6_PROMETHEUS_RW_STALE_MARKERS=true \
  grafana/k6:latest run -o experimental-prometheus-rw --tag testid=<unique-run-id> tests/k6/employees-api-stress.js
```

## Useful Overrides

- `BROWSE_RATE`: steady-state list request rate per second
- `WRITE_VUS`: concurrent writers
- `PAGE_SIZE`: page size for list requests
- `BROWSE_STAGE_ONE_DURATION`, `BROWSE_STAGE_TWO_DURATION`, `BROWSE_STAGE_THREE_DURATION`: browse ramp profile
- `WRITE_DURATION`: total write duration

## Dashboard Flow

1. Run the wrapper or the manual k6 command.
2. Open Grafana at `http://localhost:3000`.
3. Open `k6 Stress Overview`.
4. Filter the dashboard by the `testid` used for the run.
